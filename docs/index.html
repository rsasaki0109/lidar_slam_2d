slamx

CLI-first 2D LiDAR SLAM experiments, published as a lightweight public benchmark log.

slamx is a ROS-free core for offline SLAM replay, evaluation, and scan-matcher iteration. The current public slice focuses on Cartographer parity work, IILABS benchmark generalization, and a small set of visuals that are easy to reuse in the repository README.

Python 3.10+ CLI-first ROS bag support Benchmark-driven

What is here today

This site is intentionally small: one benchmark summary, one trajectory gallery, and the core commands needed to reproduce the workflow locally. The underlying repo still contains active experimental branches, but the public-facing surface stays focused on the clean story.

Backpack parity

Early Cartographer backpack_2d segments moved from meter-scale drift to sub-meter agreement by switching to Cartographer-style local matching and simpler pose graph handling.

IILABS generalization

The current front-end already beats Cartographer sampled baselines on multiple IILABS sequences. The remaining open case is ramp.

Publishing style

All visuals are generated as SVG assets so the same files work on GitHub Pages and inside the repository README.

Minimal local workflow

pip install -e .
slamx replay examples/fixture_scans.jsonl --out runs/demo --no-write-map
slamx eval ate runs/demo --gt runs/cartographer_traj_s300_window.csv

Current status

The public story is already strong enough to publish: backpack parity is substantially improved and several IILABS sequences are ahead of the reference. The one open technical thread is the ramp sequence, where pitch-robust preprocessing helps a little but does not yet close the full 2k gap.
